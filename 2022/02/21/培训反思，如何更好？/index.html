培训反思，如何更好？

问题

节奏感不强

对于时间把控的不好，准备的东西，由于前边太赶，导致很快就讲完了，后边可能没得讲了。

条理不清晰

整体讲解的过程，很混乱，react 和 es6 混合在一起，包含了各种知识点。

手写代码容易让学生不知道在干什么。

修正方向

  1. 通过把讲解内容分为 es6 和 react，避免在讲解 react 中会出现各种 es6 语法讲解的问题，
  2. 通过把时间分开，保证讲解的内容是分开的，给学生练习的时间
  3. 通过提前去提出问题，加强培训的条理性
  4. 避免现场编程，容易出现各种问题

问题修正

节奏感

  1. 9:00 - 10:00 讲解 es6
  2. 10:00 - 11:00 讲解 react

条理

  1. es6 讲解

    • let
    • 解构赋值
    • 箭头函数
    • map 方法

  2. react 讲解

    • 通过 count 案例 分析，分为多个步骤

      1. 需要一个按钮和一个 div， 在 render 中实现
      2. 需要一个 state，放置 count 状态
      3. 需要一个函数，绑定在按钮上
      4. 函数通过 this.setState 实现数据驱动

    • 通过 tab 案例 分析，分为多个步骤

      1. 通过生命周期，获取数据，处理状态
      2. render，
      3. 按钮触发当前 current
