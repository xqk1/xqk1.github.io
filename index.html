培训反思，如何更好？

问题

节奏感不强

对于时间把控的不好，准备的东西，由于前边太赶，导致很快就讲完了，后边可能没得讲了。

条理不清晰

整体讲解的过程，很混乱，react 和 es6 混合在一起，包含了各种知识点。

手写代码容易让学生不知道在干什么。

修正方向

  1. 通过把讲解内容分为 es6 和 react，避免在讲解 react 中会出现各种 es6 语法讲解的问题，
  2. 通过把时间分开，保证讲解的内容是分开的，给学生练习的时间
  3. 通过提前去提出问题，加强培训的条理性
  4. 避免现场编程，容易出现各种问题

问题修正

节奏感

  1. 9:00 - 10:00 讲解 es6
  2. 10:00 - 11:00 讲解 react

条理

  1. es6 讲解

    • let
    • 解构赋值
    • 箭头函数
    • map 方法

  2. react 讲解

    • 通过 count 案例 分析，分为多个步骤

      1. 需要一个按钮和一个 div， 在 render 中实现
      2. 需要一个 state，放置 count 状态
      3. 需要一个函数，绑定在按钮上
      4. 函数通过 this.setState 实现数据驱动

    • 通过 tab 案例 分析，分为多个步骤

      1. 通过生命周期，获取数据，处理状态
      2. render，
      3. 按钮触发当前 current

建立私有 npm 仓库，可以避免代码泄漏，方便控制 npm 源。

环境

  1. 主机(可以为本地主机)

  2. docker && docker-compose

  3. docker 虚拟机配置最低 1 CPU 和 2G 内存

  2. 然后点击 Sign in 进行登录，首次登录时，账号为 admin ，密码自动生成在 /Users/ray/nexus-data/admin.password 文件中

配置

  1. 创建 Blob Stores

  2. 创建仓库
  3. 创建 hosted 类型的仓库，选择 npm(hosted)，输入仓库名称 npm-hosted，然后点击 Create repository 创建仓库
  4. 创建 proxy 类型的仓库，选择 npm(proxy)，输入仓库名称 npm-taobao，输入代理地址 https://registry.npmmirror.com，然后点击 Create repository 创建仓库
  5. 创建 group 类型仓库，选择 npm(group)，输入仓库名称 npm-group，然后将上面创建的仓库移到 Members 中，然后点击 Create repository 创建仓库
  6. 选择 Security 下的 Realms，将 npm Bearer Token Realm 从 Available 移到 Active 中
